# english-lean — 产品与技术概览

## 1. 目标

自用、**完全离线**的桌面英语单词学习工具：以**四级左右高频词**为主，卡片呈现；**文本默写**过关后才能进入下一张；用**本地时间**驱动的**间隔复习（记忆曲线）**安排回忆。

## 2. 范围与非目标

### 2.1 包含

- 仅 **电脑端**（首版以桌面应用为目标）。
- 释义以 **中文**为主；可附音标（无语音合成、无听写音频）。
- 词条信息：**词根词缀**、**例句**、**近义词/易混词**等——**有数据则展示，无则隐藏**，不阻塞学习。
- **文本默写**：用户输入完整英文词形；**必须默写正确**才允许「下一张/划走」。
- **间隔重复**：根据复习结果更新 `next_review_at`，启动时用系统本地时间与该字段比较，生成当日复习队列。

### 2.2 不包含（当前阶段）

- 联网同步、账号、云词库。
- 语音播放、听写、发音评测。
- 移动端（可作为远期选项，不作首版承诺）。

## 3. 核心用户流程

1. 打开应用 → 合并 **到期复习词** 与 **当日新词**（数量策略可配置），形成学习序列。
2. **卡片**：展示单词；释义可先折叠，点击展开（可选，降低一眼看到中文的依赖）。
3. **展开区**（可选）：词根词缀、多义项、例句、近义词/辨析。
4. **默写区**：用户输入答案 → 校验规则见下节。
5. **错误**：提示失败，允许重试；**不**更新为「本轮成功」，**不**解锁下一张。
6. **正确**：写入复习记录 → 更新 SM-2（或选定算法）状态与 `next_review_at` → **解锁**下一张（滑动或按钮）。

## 4. 默写校验规则（建议）

- 去除首尾空白。
- **大小写策略**：统一按词表 canonical 形式比对，或对用户输入与词形同时转小写再比（与词库约定一致即可）。
- 可选：常见变体（如美式/英式拼写）若在词库中分别存为不同词条则自然分开；若同一词条多拼写需在数据层显式支持（首版可不做）。

## 5. 记忆曲线 / 间隔复习

### 5.1 原则

- 每个词条维护独立的学习状态；使用 **设备本地时间**（`datetime.now()` 或等价）判断是否到期。
- 时间存储建议：**带时区的 ISO 8601** 或 UTC 时间戳，避免歧义；若仅单机单用户，使用本地 naive datetime 也可，团队内需统一一种方式。

### 5.2 算法（首版推荐）

- **SM-2** 变体：默写正确视为高质量回忆，按间隔拉长；连续失败或长时间未复习可缩短间隔或重置 `repetitions`（细则实现时写在 `docs/plan/srs.md` 或代码注释中）。
- **简化替代**：固定阶梯天数（如 1, 2, 4, 7, 15, 30）——实现成本更低，可先做再换 SM-2。

### 5.3 状态字段（概念）

与词库 `word_id` 关联，例如：

- `ease_factor`、`interval_days`、`repetitions`
- `next_review_at`、`last_reviewed_at`
- 可选：`lapses`（失败次数）、`created_at`

## 6. 数据与离线词库

- **词库**：JSON或 SQLite 表；字段包含：`lemma`、音标、`definitions_zh`、例句、词根词缀、近义词等（均可空）。
- **词表范围**：四级核心 / 高频子集白名单，控制「只学常用」。
- **学习记录**：SQLite 推荐，便于查询「今日到期」列表。

## 7. 技术栈（待选型，倾向）

- **Python + PySide6**：桌面 + SQLite 开发路径短，适合自用。
- 备选：Tauri + 前端 + 本地 SQLite（UI 更灵活，工程量略大）。

最终选型在实现阶段写入仓库根目录 README 或 `docs/plan/tech.md`。

## 8. 风险与假设

- 用户修改系统时间会导致复习调度变化——自用可接受。
- 词根词缀、例句、近义词依赖词库质量；首版允许大量字段为空。
- 必须默写对才能下一张可能增加挫败感；保留后续「宽松模式」开关的可能（当前规格为硬要求）。

## 9. 文档维护

需求变更时同步更新本文件与 `roadmap.md`；算法细节可拆出 `srs.md` 以免本文件过长。
